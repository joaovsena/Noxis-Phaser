Noxis Phaser - Contexto de Migracao Svelte HUD
Data: 2026-03-18

OBJETIVO DESTE ARQUIVO
Este arquivo serve como contexto rapido para continuar o trabalho em outro PC, sem perder o historico do que ja foi feito no HUD Svelte e no fluxo do jogo.

RESUMO DO PROJETO
- Projeto: Noxis Phaser
- Cliente legado continua existindo
- HUD/Auth Svelte ficam isolados e so entram com query string
- Caminho legado padrao:
  http://localhost:3000
- HUD Svelte:
  http://localhost:3000/?svelteHud=1
- Auth Svelte:
  http://localhost:3000/?svelteAuth=1

CAUSA-RAIZ DO CONGELAMENTO QUE JA FOI RESOLVIDA
- O runtime Svelte continuava montado mesmo quando o HUD estava "desligado"
- O Svelte estava usando o mesmo container do UI legado (#ui-container)
- Isso fazia cliente legado e runtime Svelte concorrerem pelo mesmo estado/host visual
- Solucao aplicada:
  - criado root separado #svelte-root em client/index.html
  - mount do Svelte movido para client/src/svelte-hud/index.ts
  - em client/src/main.ts o Svelte agora so sobe com query string explicita
  - resultado: o caminho legado nao congela mais

- Reativados e integrados:

  - PartyFramesPanel

SISTEMA DE HABILIDADES - O QUE FOI CORRIGIDO
- O HUD Svelte antes tratava skills de forma simplificada e quebrada
- Problema principal:
  - hotbar Svelte so tentava castar skill em alvo de mob
  - buffs/self-casts nao funcionavam corretamente
  - auto attack aceitava skill que nao era ofensiva
- Correcao aplicada:
  - client/src/svelte-hud/stores/gameUi.ts agora diferencia skills de alvo mob e skills self-cast
  - skill self-cast pode ser usada sem alvo
  - auto-attack so aceita skills ofensivas
  - funcao para equipar skill diretamente na hotbar foi adicionada
- Redesign aplicado:
  - client/src/svelte-hud/SkillsWindow.svelte foi redesenhado
  - menu agora e vertical
  - progressao por tiers de nivel do personagem:
    - tier 1 -> nivel 1
    - tier 2 -> nivel 10
    - tier 3 -> nivel 20
    - tier 4 -> nivel 30
    - tier 5 -> nivel 40
  - cada skill mostra:
    - nivel de desbloqueio
    - rank atual
    - alvo
    - cooldown
    - motivo de bloqueio
    - acoes: Evoluir / Equipar na barra / Auto ataque / Usar agora
- Backend alinhado com a progressao:
  - server/services/SkillService.ts agora valida nivel minimo da skill
  - server/controllers/GameController.ts define tiers de desbloqueio com SKILL_UNLOCK_LEVELS
